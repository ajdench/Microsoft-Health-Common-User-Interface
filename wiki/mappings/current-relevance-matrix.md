# Current Relevance Matrix

## Purpose
This page classifies major Health CUI ideas into:
- still valid
- valid with reinterpretation
- historically interesting only

## Matrix
| Domain | Still valid | Valid with reinterpretation | Historically interesting only |
| --- | --- | --- | --- |
| Patient context | Persistent patient shell, minimum identity cluster, deceased-state differentiation | Two-zone banner composition, supplementary context placement | Literal fixed-width geometry and hover-era behaviors |
| Medication display | Explicit dose salience, stable attribute order, no silent truncation | Exact typography, label styling, dense desktop wrapping patterns | Literal visual examples and platform-era control assumptions |
| Search and prescribe | Progressive narrowing, review-before-authorise, constrained selection for risky attributes | Prioritised result sections, cascading lists, keyboard accelerators | Stack-specific control mechanics from legacy desktop/web forms |
| Alerts and decision support | Capability status, grouped priorities, incomplete-coverage messaging | Pop-up summaries, exact information-window anatomy, two-level priority vocabulary | Exact iconography and exploratory visual proposals |
| Data manipulation | Visible subset state, clear filter/sort/group status, reversible transformations | Specific toolbar/control arrangements, progressive sort details | Desktop-era visual analogies and form-factor assumptions |

## Notes
- `Still valid` means the idea remains conceptually strong for current clinician-facing systems.
- `Valid with reinterpretation` means the idea is useful, but the historical interaction pattern should be modernized for accessibility, responsiveness, and current governance.
- `Historically interesting only` means the idea is useful mainly for understanding programme history or design rationale, not for direct adoption.
