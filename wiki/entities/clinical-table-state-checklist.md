# Clinical Table State Checklist

## Use this when
Designing or reviewing filters, sorting, grouping, and other stateful data views in clinician-facing tables and lists.

## Checklist
- Always show when the current view is filtered, sorted, grouped, or otherwise transformed.
- Make the active subset or manipulation state visible near the data, not only inside a hidden drawer.
- Provide a clear path to remove filters and reset sorts/groups.
- Do not imply that a filtered subset represents the full patient record.
- Preserve result counts or other cues that help users judge scope.
- Keep sort direction explicit and reversible.
- Make grouped views clearly labeled so clinicians know the organizing principle.
- Preserve keyboard efficiency for repeated high-frequency data tasks.
- Avoid generic enterprise-grid defaults that hide state behind small icons or overflow menus.
