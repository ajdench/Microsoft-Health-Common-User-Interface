## Trustworthiness / caveats
- This is a preservation host, not the original CodePlex service.
- It is strong evidence for release chronology, package naming, and license publication.
- It should not be treated as proof that every original release asset survived unchanged.
- The surviving source-code tab is thin and currently exposes only build-process template files.
- Direct attachment URLs for release readmes and packages currently return authentication errors, so the archive is useful but incomplete.

## Short summary
The recovered CodePlex archive closes a major provenance gap. It provides a project landing page, a detailed releases page, and the published license text.

## Key extractable insights
- The archive exposes a release history from `1.0.108.004` and `1.0.114.000` on 13 July 2007 through `8.2.51.000` on 28 May 2010.
- The releases page records a version-number jump from `1.5` to `6.4`, explicitly saying the team aligned internal and external version numbers.
- The `1.3` release notes say legacy AJAX medication controls were deprecated and replaced with a new WPF/Silverlight `MedicationsListView`.
- The archive confirms publication under the Microsoft Public License (`Ms-PL`).
- The surviving source-code tree currently shows only `BuildProcessTemplates`, which suggests the archive is not exposing a complete browseable source tree.

## Should this influence modern reinterpretation?
Yes for chronology, licensing, and understanding platform shifts. No for direct platform choices such as WPF/Silverlight migration.
